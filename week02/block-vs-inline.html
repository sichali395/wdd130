Task3: Nesting Block in Inline

Try placing a <div> inside a <span> element.

What error or layout issue do you see? Why does this happen?

Placing a block-level element inside an inline-level element is invalid HTML according to the specification. The main issue you will see is that the browser will break the nesting. It will effectively "close" the “span” tag before the “div” starts. This happens because the content model for an inline element like “span” is only supposed to contain other inline elements. When the browser's parser encounters a block element where it shouldn't be, it corrects the invalid markup to create a valid document structure. Visually, this means the “div” will not be contained inside the “span”. The “div” will break onto a new line (as it is a block element) and appear after the now-closed “span”, completely breaking the intended layout.

Task 4: Review the HTML markup of the following Inline Elements

<img> element

1. The image is an inline element (more specifically, a replaced inline element). It sits within the flow of text. You can see this because if you put text before and after it in the HTML, the text would appear on the same line as the image's bottom edge. However, you can set dimensions on it (as shown with width="300" height="300"), which is not typical for non-replaced inline elements like “span”.

2. The anchor tag is also an inline element. By default, it is displayed in blue, underlined text and only takes up the width of its content. It flows within a line of text without causing breaks.

3. The button is an inline element. It behaves like other inline elements, flowing within the line of text. However, it has a default width, height, and styling (provided by the operating system or user agent stylesheet) that makes it look like a interactive button.
